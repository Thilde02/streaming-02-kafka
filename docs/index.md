## Custom Project


Dataset

My Kafka producer uses the dataset:

File name: sales.csv

Location: data/ folder

Type of records: Simulated retail sales transactions

Fields included:  
order_id, customer_id, product_id, quantity, unit_price,
currency_code, region_id, datetime, device_type,
payment_method, referral_source, discount_code,
is_online, is_new_customer, customer_note

Dataset used: I used the original dataset without modifying any records.

Kafka Messages
Producer used: kafka_producer_case.py

What the producer sends:  
Each message is one row from the sales.csv dataset, converted into a dictionary.

Kafka topic:  
product-sales-case

Message key:  
The region_id field is used as the Kafka message key.

Message fields:  
All original fields from the dataset are included. No fields were removed or changed.

Consumer Processing
My custom consumer file is:

Code
kafka_consumer_tiffany.py
The consumer:

Connects to Kafka

Reads all available messages from the topic

Logs each message

Processes each message

Writes the processed message to:

Code
src/data/output/consumed_sales_tiffany.csv
Phase 5 Enhancement: Added a New Field
I added a new calculated field called order_total, which multiplies:

Code
unit_price × quantity
This enriches each message with the total dollar value of the order.

What the consumer receives:
A full sales transaction record from Kafka

All original fields

The new order_total field added during processing

How many messages it consumed:
My consumer processed 6 messages from the topic.

What it logs:
Each raw message

Results
When I ran the producer and then my custom consumer:

Kafka successfully streamed all 6 messages

My consumer processed each message

The new order_total field appeared in the output CSV

The logs confirmed successful execution

The pipeline ran cleanly from start to finish

Interpretation
This Kafka workflow showed me:

How data flows from a CSV → producer → Kafka topic → consumer → output CSV

How easy it is to enrich streaming data with new fields

How message keys (like region_id) can support partitioning and analysis

How consumers can be customized for business logic

Business insight:  
The new order_total field helps identify high‑value orders, track revenue, and support real‑time analytics.
A business could use this stream to monitor sales performance, detect trends, or trigger alerts for large orders.
